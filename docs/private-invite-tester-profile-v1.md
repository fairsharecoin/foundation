# FSC Private Invite Tester Profile v1

Date: 2026-04-07
Owner: Denis + Echo

## Cohort Size
- Initial cohort: 3–8 trusted testers (friends)
- Expand only after first issue-triage cycle is stable

## Tester Selection Criteria
1. Trust level: known personally by Denis
2. Willing to test carefully and report clear repro steps
3. Comfortable with experimental software and rough edges
4. Accepts that this is alpha (not production-ready)

## Access Method (initial)
- Invite by direct message only
- Accounts provisioned intentionally (no open signup blast)
- Access can be revoked quickly if misuse/noise occurs

## What Testers Should Focus On
1. Onboarding/login/session behavior
2. Wallet overview clarity
3. Transfer flow (UUID-only path)
4. Recipient management and history views
5. Error quality (is it understandable and safe)

## What Testers Should NOT Assume
- No production guarantees
- No final UX polish
- No promise of permanent data retention during alpha

## Reporting Format (minimum)
- What you did
- What you expected
- What happened
- Timestamp + screenshot (if possible)

## Stop Conditions
Pause cohort expansion if any of the following appear:
- Data integrity concern
- Security/privacy concern
- Reproducible transfer inconsistency
